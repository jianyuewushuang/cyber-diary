# 赛博日记本

一个可视化日记管理工具，将本地 Markdown 格式的日记文件转换为精美的网页，支持日历视图、搜索和数据统计功能。


## 使用方法

### 1. 添加日记文件

在 `diary/` 文件夹中创建 Markdown 格式的日记文件，**文件名必须遵循以下格式**：

**格式一**：
```
yyyyMMdd.md
```

例如：
- `20260606.md`（2026年6月6日）
- `20260705.md`（2026年7月5日）

**格式二**：
```
yyMMdd.md
```

例如：
- `260606.md`（自动转换为2026年6月6日）
- `260705.md`（自动转换为2026年7月5日）

> **提示**：6位日期格式会自动添加 `"20"` 前缀，转换为 `20xx` 年的日期。

**注意**：
- 文件名必须是纯数字的日期格式
- 文件内容使用标准 Markdown 语法
- 支持标题、列表、加粗、链接等 Markdown 特性

> 我在项目根目录准备了`start.sh`，linux用户可在添加日记文件后直接运行这个脚本启动服务。

### 2. 构建生成 HTML

执行构建命令，将日记文件转换为网页：

```bash
npm run build
```

构建成功后会输出：
```
找到 X 篇日记
HTML 文件已生成到 build 文件夹
```

生成的 HTML 文件位于 `build/index.html`。

### 3. 预览网页

**方式一：使用开发服务器**

```bash
npm run dev
```

或手动启动：

```bash
npx http-server build -p 8081
```

然后在浏览器访问：`http://localhost:8081` 或 `http://127.0.0.1:8081`

**方式二：直接打开文件**

直接双击打开 `build/index.html` 文件（部分功能可能受限）。

### 示例日记内容

```markdown

今天天气很好，阳光明媚。
```

## 功能介绍

### 1. 日历视图
- 以日历形式展示所有日记
- 有日记的日期会显示紫色方块标记
- 方块大小根据日记字数动态变化（字数越多，方块越大）
- 点击日期可查看日记详情

### 2. 日记详情面板
- 右侧弹出详情面板（宽度 500px）
- 支持 Markdown 渲染
- 显示日记字数统计

### 3. 搜索功能
- 顶部搜索框支持按内容搜索
- 搜索结果显示日记预览（200字）
- 实时匹配高亮

### 4. 统计图表（折线图）
- **每日统计**：按日期统计字数
- **每周统计**：按周统计日记数量和字数
- **每月统计**：按月统计日记数量和字数
- **每年统计**：按年统计日记数量和字数
- 无日记的日期显示为 0
- 可点击对应小圆点快速跳转到对应日记

### 5. 导航功能
- 年份/月份下拉选择器
- 上一月/下一月快捷切换
- 支持快速跳转到任意年月

## 常见问题

### Q：如何更新日记？
A：修改 `diary/` 文件夹中的 Markdown 文件后，重新运行 `npm run build` 即可更新网页。

### Q：为什么某些日期没有显示？
A：系统会自动填充第一篇和最后一篇日记之间的所有日期。如果日期超出这个范围，则不会显示。

### Q：搜索结果预览字数可以调整吗？
A：可以在 `index.js` 第38行修改预览字数。

### Q：如何删除日记？
A：直接删除 `diary/` 文件夹中对应的 `.md` 文件，重新构建即可。

## 项目背景

### 创意介绍

在平时写日记时发现写了很多日记之后很难直观感受写好的日记，每次点开查看太麻烦，于是便想做一个可视化赛博日记本，方便查看日记和统计日记数据。

产品是一个命令行工具，通过几条简单的命令把仓库中的日记做成精美的 HTML 预览、查看和统计页面。

### 目标用户及痛点

- 面向经常写电子日记，并追求对文件的掌握权的用户
- 当用户不想用复杂的笔记软件进行日记管理，追求完全自主掌握日记文件，而又苦于难以对大量日记文件进行整理统计和快速查看时，这个产品会发挥很大作用
- 如果没有这个产品，用户对本地日记文件的管理非常不方便，难以做到直观可视化感知，想查看和搜索日记也非常困难

### 价值与意义

这款产品可以帮助用户管理日记，让日记对用户生活的记录发挥更大意义。
